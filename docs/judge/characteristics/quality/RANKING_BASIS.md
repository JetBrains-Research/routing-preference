<BASIS>
Rank solutions from best to worst code quality.

Higher rank — solution is cleaner and more maintainable:
- Clean, well-structured, easy to follow
- Clear naming and organized logic
- Complex parts are documented
- Style is consistent with the codebase's conventions

Lower rank — solution has quality issues:
- Minor issues such as an unclear variable name, missing comment on a non-obvious block, or small deviation from conventions
- Noticeable gaps: logic that takes effort to follow, missing documentation on complex sections, inconsistent naming or formatting
- Significant problems that make the code hard to read, understand, or maintain
- Largely incomprehensible or unmaintainable: severely inconsistent formatting, total absence of documentation on complex logic, deeply unclear naming, or structure that obscures rather than communicates intent

Tiebreakers when two solutions appear equally clean:
1. Prefer the solution with better adherence to the codebase's conventions
2. Prefer the solution that is easier for a reviewer to follow and maintain

Note: you are ranking CODE QUALITY, not correctness or scope. A buggy but clean solution still ranks higher here than a correct but poorly written one.
</BASIS>
