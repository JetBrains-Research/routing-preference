Code quality evaluates the standard of the entire solution across four dimensions:

- Readability: is the code easy to read and understand at a glance?
- Maintainability: is the code structured in a way that is easy to modify or extend?
- Documentation: are complex or non-obvious parts of the solution explained?
- Convention adherence: does the solution follow the correct naming, formatting, and structural conventions?

All four dimensions are evaluated together as a whole.

Note: Whether the solution works should not be evaluated. Whether it stays within scope should not be evaluated. This characteristic is purely about how well the solution is written.
</LONG_DESCRIPTION>
